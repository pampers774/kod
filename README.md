# ПЛАН ПРОГРАММЫ: Ежедневник с тёмной темой

## 1. Общее назначение

Программа представляет собой локальное приложение‑ежедневник с графическим интерфейсом, позволяющее пользователю:


* вести записи в разных дневниках;
* управлять дневниками и записями;
* просматривать, добавлять и удалять записи;
* работать в удобной тёмной теме.

Все данные хранятся локально на компьютере, без подключения к интернету.

## 2. Основные компоненты

### 2.1. База данных

Программа использует SQLite‑файл для хранения данных. В нём содержатся:


* список дневников;
* все записи с датами и привязкой к дневникам.

**Таблицы:**

* **Дневники** — хранит имена дневников. Имя должно быть уникальным.
* **Записи** — хранит текст, дату и ссылку на дневник. При удалении дневника автоматически удаляются все его записи.

**При первом запуске:**

* создаются необходимые таблицы;
* добавляется дневник по умолчанию — «Основной».

### 2.2. Графический интерфейс (GUI)

Окно программы выполнено в тёмной цветовой схеме для удобства чтения и эстетики. Интерфейс разделён на логические блоки.

**Основные элементы:**

* **Область просмотра записей** — отображает все записи с именами дневников и датами. Новые записи показываются сверху.
* **Выпадающий список дневников** — позволяет выбрать, в какой дневник добавлять новую запись.
* **Поле ввода текста** — многострочное поле для написания новой записи.
* **Панель кнопок** — содержит все действия пользователя.

## 3. Функциональные возможности


### 3.1. Работа с дневниками


* **Просмотр списка дневников** — отображается в выпадающем меню.
* **Добавление нового дневника** — через всплывающее окно с вводом имени. Проверяется, что имя не дублируется.
* **Удаление дневника** — возможно только для дневников, кроме «Основного». Удаление влечёт удаление всех записей в нём.

### 3.2. Работа с записями

* **Создание записи** — текст сохраняется в текущий дневник с автоматической датой и временем.
* **Просмотр всех записей** — отображается в одном окне с разбивкой по дневникам и датам.
* **Удаление записи** — через отдельное окно, где можно выбрать дневник и конкретную запись из списка.

### 3.3. Отображение данных

Все записи показываются в хронологическом порядке — от новых к старым.

Каждая запись отображается с:

* иконкой и именем дневника (цветом);
* датой (серым курсивом);
* текстом записи.

Если записей нет — выводится информационное сообщение.

## 4. Цветовая схема (тёмная тема)

Интерфейс использует тёмные тона для снижения нагрузки на глаза.

**Основные цвета:**

* глубокий чёрно‑серый фон;
* светлый текст для читаемости.

**Акцентные цвета для кнопок:**

* зелёный — для сохранения;
* синий — для нейтральных действий;
* красный — для удаления.

## 5. Логика работы программы

### При запуске:

1. Проверяется наличие базы данных.
2. При необходимости — создаются таблицы.
3. Если дневников нет — добавляется «Основной».
4. Загружается главное окно с интерфейсом.

### Основной цикл:

1. Пользователь выбирает дневник.
2. Пишет запись в поле ввода.
3. Нажимает «Сохранить» — запись добавляется с датой.
4. Отображение обновляется автоматически.

### Дополнительные действия:

* **Обновить** — перезагружает список дневников (на случай, если были изменения).
* **Просмотреть все** — показывает все записи в одном окне.
* **Новый дневник** — добавляет категорию для записей.
* **Удалить запись** — через отдельное окно с выбором.
* **Удалить дневник** — с подтверждением и защитой «Основного».
